# E-Commerce-Analysis
Sales analysis of 100K+ orders using Python and SQL

<b> About This Project </b>
This is an end-to-end data analysis project where I have analyzed a real-world e-commerce dataset from a Brazilian company called <b> Olist </b>

The dataset contains <b> 100,000+ orders </b> placed between <b> 2016 and 2018 </b> across 9 relational tables. I used Python to load and clean the data, SQL to answer important business questions, and Matplotlib and Seaborn to visualize the results.

This project demonstrates my ability to work with real-world messy data, write SQL queries from basic to advanced level, insights in a clear and visual way.

   <b> Questions Answered </b>
   <br>
   <br>
   <b> Basic SQL Queries </b>
   1. What is the total revenue earned each month?
   2. Which are the top 10 sellers by revenue?
   3. Which product categories generate the most sales?
   4. Which states have the highest number of customers?
   5. How are orders distributed by their status?
   6. Which product categories have the best review scores?

<b> Advanced SQL Queries </b>

   7. What is the month over month revenue growth percentage?
   8. How do customers rank based on their total spending?

   <hr>

   <b> Key Findings </b>
   <ul>
      <li> Revenue grew steadily from 2016 to 2018 with a strong peak in <b> November </b>(Black Friday effcet)</li>
      <li> <b> Health and Beauty </b> is the top revenue generating product category</li>
      <li> <b> Sao Paulo (SP) </b> has the highest number of customers in Brazil </li>
      <li> Over <b> 96% of orders </b> were successfully delivered </li>
      <li> Most product categories maintain a review score above <b> 4 out of 5 </b> </li>
      <li> Majority of customers are <b> one-time-buyers </b> showing a high churn rate </li>
   </ul>
